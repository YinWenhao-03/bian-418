# 交易机器人监控系统 - 快速启动指南

## 一、系统要求
- Node.js 16+
- MySQL 5.7+ 或 8.0+

## 二、快速开始

### 1. 解压文件
```bash
unzip trading-bot-monitoring-mysql.zip
cd trading-bot-monitoring-mysql
```

### 2. 配置环境变量
复制环境变量示例文件：
```bash
cp .env.example .env
```

编辑 `.env` 文件，配置以下内容：

```env
# 币安 API 配置
BINANCE_API_KEY=你的币安API密钥
BINANCE_API_SECRET=你的币安API密钥密码

# MySQL 数据库配置
MYSQL_HOST=localhost
MYSQL_PORT=3306
MYSQL_USER=root
MYSQL_PASSWORD=你的MySQL密码
MYSQL_DATABASE=trading_bot
```

### 3. 创建 MySQL 数据库
登录 MySQL：
```bash
mysql -u root -p
```

创建数据库：
```sql
CREATE DATABASE IF NOT EXISTS trading_bot DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
exit;
```

### 4. 安装依赖
```bash
npm install
```

### 5. 启动应用
```bash
npm run dev
```

这将同时启动：
- 前端开发服务器（默认 http://localhost:5173）
- 后端 API 服务器（默认 http://localhost:3000）

### 6. 访问应用
打开浏览器访问：http://localhost:5173

## 四、API 接口快速参考

### 机器人控制
- `GET /api/bot/status` - 获取机器人状态
- `POST /api/bot/start` - 启动机器人
- `POST /api/bot/stop` - 停止机器人
- `POST /api/bot/reset` - 重置机器人状态

### 交易历史
- `GET /api/trades` - 获取交易历史（支持筛选和分页）
- `GET /api/trades/:id` - 获取单个交易详情

### 系统日志
- `GET /api/logs` - 获取系统日志（支持筛选和分页）
- `DELETE /api/logs/old` - 删除旧日志

详细的 API 文档请参考 `配置说明.md` 文件。

## 五、注意事项

1. **币安 API 安全**
   - 妥善保管您的 API 密钥
   - 不要将 `.env` 文件提交到版本控制
   - 建议在币安设置 IP 白名单

2. **MySQL 数据库**
   - 确保 MySQL 服务正在运行
   - 确保数据库用户有足够的权限
   - 建议定期备份数据库

3. **风险提示**
   - 本系统仅供学习和测试使用
   - 真实交易存在风险，请谨慎使用
   - 建议先用小额资金测试

4. **日志管理**
   - 日志会不断增长，建议定期清理旧日志
   - 可以通过 DELETE /api/logs/old 接口删除旧日志

## 六、故障排除

### 数据库连接失败
- 检查 MySQL 服务是否启动
- 检查 `.env` 中的数据库配置是否正确
- 确认数据库用户有足够权限

### 端口被占用
- 修改 `vite.config.ts` 或后端配置更改端口
- 或者关闭占用端口的其他程序

### 币安 API 错误
- 检查 API 密钥是否正确
- 确认 API 密钥有足够的权限
- 检查网络连接和防火墙设置

详细的配置和使用说明请参考 `配置说明.md` 文件。
