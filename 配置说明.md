# 交易机器人监控系统 - 配置说明

## 一、环境配置

### 1.1 复制环境变量文件
```bash
cp .env.example .env
```

### 1.2 配置币安 API
在 `.env` 文件中配置您的币安 API 密钥：
```env
BINANCE_API_KEY=your_actual_api_key
BINANCE_API_SECRET=your_actual_api_secret
```

### 1.3 配置 MySQL 数据库
在 `.env` 文件中配置 MySQL 数据库连接信息：
```env
MYSQL_HOST=localhost
MYSQL_PORT=3306
MYSQL_USER=root
MYSQL_PASSWORD=your_mysql_password
MYSQL_DATABASE=trading_bot
```

## 二、数据库初始化

### 2.1 创建数据库
首先在 MySQL 中创建数据库：
```sql
CREATE DATABASE IF NOT EXISTS trading_bot DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

### 2.2 表结构说明
系统会自动创建以下表：

#### trades 表（交易记录）
- 存储所有交易历史记录
- 字段：id, symbol, side, price, quantity, timestamp, status, profit_loss, order_id, fee, note, extra

#### logs 表（系统日志）
- 存储系统运行日志
- 字段：id, level, message, timestamp, metadata, source, user_id, extra

#### bot_status 表（机器人状态）
- 存储机器人运行状态
- 字段：id, status, started_at, last_updated, trade_count, strategy, version, extra


### 3.2 交易历史 API

#### 获取交易历史
```
GET /api/trades?limit=100&offset=0&status=completed&symbol=BTCUSDT&startTime=1234567890&endTime=1234567899
```
查询参数说明：
- `limit`: 返回记录数（可选，默认 100）
- `offset`: 偏移量（可选，用于分页）
- `status`: 按状态筛选（可选：pending, completed, failed, cancelled）
- `symbol`: 按币种筛选（可选）
- `startTime`: 开始时间戳（可选）
- `endTime`: 结束时间戳（可选）

响应示例：
```json
{
  "success": true,
  "data": [
    {
      "id": 1,
      "symbol": "BTCUSDT",
      "side": "buy",
      "price": 59800.00,
      "quantity": 0.001,
      "timestamp": 1234567890,
      "status": "completed",
      "profit_loss": null,
      "order_id": "123456789",
      "fee": null,
      "note": null,
      "extra": null
    }
  ]
}
```

#### 获取单个交易详情
```
GET /api/trades/:id
```
响应示例：
```json
{
  "success": true,

    "id": 1,

```

### 3.3 系统日志 API

#### 获取系统日志
```
GET /api/logs?limit=100&offset=0&level=info&source=TradingService&startTime=1234567890&endTime=1234567899
```
查询参数说明：
- `limit`: 返回记录数（可选，默认 100）
- `offset`: 偏移量（可选，用于分页）
- `level`: 按级别筛选（可选：debug, info, warn, error, fatal）
- `source`: 按来源筛选（可选）
- `startTime`: 开始时间戳（可选）
- `endTime`: 结束时间戳（可选）

响应示例：
```json
{
  "success": true,
  "data": [
    {
      "id": 1,
      "level": "info",
      "message": "交易机器人已启动",
      "timestamp": 1234567890,
      "metadata": null,
      "source": "TradingService",
      "user_id": null,
      "extra": null
    }
  ]
}
```

#### 删除旧日志
```
DELETE /api/logs/old?beforeTime=1234567890
```
查询参数说明：
- `beforeTime`: 删除此时间戳之前的日志（必需）

响应示例：
```json
{
  "success": true,
  "data": {
    "message": "Old logs deleted successfully"
  }
}
```

### 5.2 新增页面功能

#### 监控页面 (/monitor)
- 显示机器人当前状态（运行/停止/错误）
- 显示运行时间、已执行交易数、启动时间
- 提供启动/停止/重置按钮
- 每5秒自动刷新状态

#### 交易历史页面 (/trade-history)
- 表格显示所有交易记录
- 支持按状态和币种筛选
- 支持分页显示

#### 系统日志页面 (/logs)
- 列表显示系统运行日志
- 不同级别用不同颜色区分
- 支持按级别和来源筛选
- 支持分页显示

## 六、注意事项

1. **币安 API 安全**
   - 妥善保管您的 API 密钥
   - 不要将 `.env` 文件提交到版本控制系统
   - 建议在币安设置 IP 白名单

2. **MySQL 数据库**
   - 确保 MySQL 服务正在运行
   - 确保数据库用户有足够的权限
   - 建议定期备份数据库

3. **风险提示**
   - 本系统仅供学习和测试使用
   - 真实交易存在风险，请谨慎使用
   - 建议先用小额资金测试

4. **日志管理**
   - 日志会不断增长，建议定期清理旧日志
   - 可以通过 DELETE /api/logs/old 接口删除旧日志
